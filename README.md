# Quinbay Projects

This repository showcases a collection of projects and modules developed as part of professional experience at Quinbay Technologies (Blibli) E-commerce. It demonstrates expertise in building scalable applications, working with microservices, processing data, and integrating APIs.

---

## Overview

The projects in this repository include:

- Backend services using Java and Spring Boot
- API integrations (internal and external)
- Data processing and analytics
- Task scheduling and automation
- Modular architecture for maintainability and scalability

---

## Key Features

- **Modular Design**: Each project/module is self-contained and reusable.
- **Data Processing**: Efficient handling of large datasets and batch operations.
- **API Management**: Building and consuming RESTful APIs.
- **Scalability & Reliability**: Designed with performance and maintainability in mind.

---

## Technologies Used

- Java 11+
- Spring Boot
- Maven / Gradle
- H2 / other relational databases
- Apache POI (for Excel processing)
- iText / PDFBox (for PDF generation)
- Git and GitHub for version control

---


## Repository Structure

Here’s an overview of the main directories and their purpose:

- **bulk-processor** – Handles batch data processing operations.
- **graphics-processor** – Manages image and graphics processing tasks.
- **module-domain-product** – Core module for product domain logic.
- **pcu-external-api** – External APIs for PCU (Product Control Unit) integrations.
- **pcu-internal-api** – Internal APIs for PCU communication between modules.
- **pcu-master-api** – Master API service for orchestrating PCU operations.
- **product-analytics** – Analytics services and reporting for product data.
- **product-category-base** – Base definitions for product categories.
- **product-distribution-task** – Task scheduler and manager for product distribution.
- **product-service** – Service layer for handling product-related operations.
- **product** – Core product module containing models and business logic.
